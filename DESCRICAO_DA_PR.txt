# Hugging Face Reachy Mini — MuJoCo & Webots Simulation Bridge for Fabric RoboPay

> **Submitted for the Hugging Face Reachy Mini Tier 1 — Simulator Skill Execution Bounty** 🤖🎉

## Overview
This PR implements a full **Fabric RoboPay simulation bridge (`mujoco_sim_bridge_reachy_mini`)** for the **Hugging Face Reachy Mini** robot. Following the architectural pattern of `bridge/unitree/g1`, this bridge subscribes to the `robot/tunnel/action` Zenoh topic, executes an expressive **object-tracking task** using the official Reachy Mini MuJoCo & Webots models, and publishes structured head-tracking metrics back on `robot/reachy_mini/metrics`.

## Key Architectural & Technical Highlights

### 1. Official Hugging Face / Pollen Robotics MJCF & Webots VRML Scenes
- Loads the official MuJoCo scene directly from the installed `reachy-mini[mujoco]` package (`descriptions/reachy_mini/mjcf/scenes/minimal.xml`).
- Includes a VRML Webots scene (`webots_project/worlds/reachy_mini_tabletop.wbt`) matching the tabletop layout (apple, croissant, duck, table).
- Uses authentic 3D STL meshes, realistic mass/inertia tensors, and accurate joint limits for the 6-DOF Stewart parallel neck platform and `yaw_body` torso rotation.

### 2. Expressive Social Skill Execution (`object_tracking`)
Since the Reachy Mini is an **expressive social robot without arms**, general pick-and-place commands are gracefully mapped to an **active object-tracking & expressiveness skill**:
- **Finite-State Machine (FSM):** `SCANNING` ➔ `TRACKING` ➔ `EXPRESSIVE` ➔ `DONE`
- **Slew-Rate Limiting & Low-Pass Filtering:** Ensures smooth, natural, physically realistic servo velocities.
- **Celebration Dance (`EXPRESSIVE`):** Once locked onto the target (`apple`, `croissant`, or `duck`), the robot performs animated head movements and antenna gestures.

### 3. Dual-Engine Cross-Simulator Sim-to-Sim Validation (`MuJoCo` ➔ `Webots`)
Includes a multi-simulator validation runner (`sim2sim.py`) that evaluates policy generalization across **both MuJoCo and Webots physics engines**:
- **Run 1 (MuJoCo):** Randomized friction & mass perturbations.
- **Run 2 (Webots):** Real Webots physics engine execution across 3 target objects (`apple`, `croissant`, `duck`).
- **Run 3 (MuJoCo):** Multi-target tracking verification.

### 4. 100% Package Structure Compliance
Fully follows ROS2 `ament_python` conventions while retaining standalone cross-platform Python support:

```text
bridge/reachy_mini/
├── README.md
├── test_publisher.py               # Local Zenoh ActionEvent simulator
└── mujoco_sim_bridge/
    ├── main.py                     # Standalone bridge entrypoint
    ├── visualize.py                # 3D interactive real-time visualizer
    ├── reachy_mini/
    │   ├── node.py                 # ReachyMiniBridgeNode (Zenoh pub/sub)
    │   └── mapper.py               # Action mapping (look_at, wave, track -> object_tracking)
    └── src/
        ├── policy/
        │   └── controller.py       # ReachyTaskPolicy (4-phase FSM + motor filter)
        └── simulation/
            ├── environment.py      # ReachyMiniEnvironment (MuJoCo wrapper)
            ├── webots_env.py       # ReachyMiniWebotsEnvironment
            ├── metrics.py          # SimulationMetricsTracker (angular error, FOV lock)
            ├── sim2sim.py          # Sim2SimValidator (MuJoCo + Webots cross-validation)
            └── webots_project/
                ├── worlds/reachy_mini_tabletop.wbt
                └── controllers/reachy_mini_controller/
```
