## Reviewer-requested payment-to-execution guarantees

The paid execution path is now fully execution-gated:

```text
Fabric Gateway
    ↓
x402 payment verification
    ↓
robot/tunnel/action
    ↓
Reachy Mini bridge + MuJoCo execution
    ↓
robot/tunnel/result
    ↓
x402 settlement only after SUCCESS
```

### Payment and failure guarantees

- The real Go Tunnel waits for the correlated `robot/tunnel/result`.
- Settlement occurs only after `execution_status: SUCCESS`.
- Simulator failure returns HTTP `502`; settlement is skipped.
- Simulator timeout returns HTTP `504`; settlement is skipped.
- Invalid or unpaid requests return HTTP `402` and publish no `ActionEvent`.
- Replay requests are rejected with HTTP `409`.
- Results are correlated using `action_id`, `robot_id`, `skill_id`,
  `params_hash`, and `idempotency_key`.
- `robot/reachy_mini/metrics` remains available for compatibility telemetry.
